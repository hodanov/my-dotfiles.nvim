# Amendment: supply-chain-hardening — 2026-04-03_001

## 変更箇所

`Phase 4: 検証` > `4-1. GHA ピン留めの検証`

## 変更内容

### Before

```bash
grep -rh 'uses:' .github/workflows/ \
  | sed 's/.*uses: *//' \
  | grep -v '#' \
  | grep -v '^\.\/' \
  | wc -l
# → 0 であること

pinact verify .github/workflows/*
```

### After

```bash
grep -rn 'uses:' .github/workflows/ \
  | grep -v '@[a-f0-9]\{40\}' \
  | grep -v '^\.\/'
# → 0 行であること
```

> **注意**: `@branch` や `@tag` で意図的に参照している Action（自己参照など）がある場合は、
> リポジトリに合わせて追加の `grep -v` で除外すること。除外パターンはユーザーに確認する。

## 変更理由

- `grep -v '#'` はコメントの有無で判定するため、コメントなしの SHA ピン留め済み Action を誤検出する
- `@[a-f0-9]{40}` で SHA の有無を直接判定することで正確にカウントできる
- `pinact verify` は一部バージョンで未実装のため削除
- 自己参照など意図的な `@branch` 参照は汎用スキルとして除外パターンをハードコードせず、注記のみ記載

## 期待される効果

- 検証コマンドの誤検出ゼロ
- コメントなし SHA ピン留めのリポジトリでも正確に動作

## リスク

- `grep -rn` に変更したため出力に行番号が付く（カウントには `wc -l` を別途使う必要あり）
